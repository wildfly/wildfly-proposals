= [WFCORE-7450] service.bat: Support for startup mode "delayed"
:author:            Lukas Vydra
:email:             lvydra@redhat.com
:toc:               left
:icons:             font
:idprefix:
:idseparator:       -

== Overview

The WildFly `service.bat` script allows installing WildFly as a Windows service. Currently, the `/startup` parameter supports only configuring manual or auto service startup mode. "delayed" startup mode option is not supported. As the delayed startup mode is a standard Windows service feature that delays automatic service startup until after other automatic services have started, it should be supported.

This enhancement clarifies and eventually adds support for all three Windows service startup modes (manual, auto, delayed) in the `service.bat` script, ensuring that the delayed mode is properly validated and documented.

=== User Stories

User want to configure WildFly as a delayed auto-start service so that it starts automatically after critical dependencies (such as database services, network infrastructure, and other foundational services) have fully initialized. This ensures WildFly has all required resources available at startup and reduces the risk of startup failures due to missing dependencies.

== Requirements

* The `service.bat` script should support setting the Windows service startup mode to "delayed" (Automatic - Delayed Start).
* The default startup mode when no `/startup` parameter is provided should remain "manual" to maintain backwards compatibility.
* When `/startup` is specified without a value, startup mode should be set to "auto" for backwards compatibility.
* The `/startup` parameter should accept three values: manual, auto, or delayed (case-insensitive).
* Invalid startup mode values should be rejected with a clear error message.
* The implementation must pass the startup mode value to `prunsrv.exe` via the `--Startup` parameter.

=== Non-Requirements

=== Future Work

== Backwards Compatibility

This feature maintains full backwards compatibility with existing installations:

* The default startup mode when no `/startup` parameter is provided remains "manual".
* When `/startup` is specified without a value, the startup mode defaults to "auto" as it does currently.
* Existing service installations will continue to function without changes.

== Release Note Content

WildFly's Windows service installation script (`service.bat`) now supports all three Windows service startup modes: Manual, Automatic, and Automatic (Delayed Start). Use `service.bat install /startup delayed` to configure the service to start automatically after other automatic services have completed initialization.
