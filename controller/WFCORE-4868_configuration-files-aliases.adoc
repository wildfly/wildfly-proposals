== Overview

Introduce aliases for standard configuration files

This will allow users to provide a short alias instead of a long filename, e.g. `standalone.sh -c full` instead of `standalone.sh -c standalone-full.xml`

.Full list of aliases:
|===
|  Configuration file | Aliases

| *-full.xml             | full

| *-full-ha.xml          | fullha, full-ha

| *-ha.xml               | ha

| *-load-balancer.xml    | lb

| *-microprofile.xml     | mp, micro

| *-microprofile-ha.xml  | mpha, micro-ha
|===

== Requirements

=== Hard Requirements

Allow the commandline parameter `-c` to work with an alias of a configuration file instead of a full filename.
Provide aliases for all standard configuration files (except the default).

=== Nice-to-Have Requirements

Allow users to leverage the alias for their custom configuration files.
This would only work for files named `standalone-<alias>.xml`.
