= Promote OIDC Logout Support to Default Stability
:author:            Diana Krepinska
:email:             dvilkola@ibm.com
:toc:               left
:icons:             font
:idprefix:
:idseparator:       -

== Overview

OIDC logout support (RP-Initiated Logout, Front-Channel Logout, and Back-Channel Logout) is currently available under PREVIEW stability.
This analysis proposes promoting the feature to DEFAULT stability so the functionality is available in the standard WildFly distribution and defaults.

=== User Stories

* A user secures applications with `elytron-oidc-client` and expects a complete logout flow (local application logout plus provider-driven session termination) without requiring preview-only server operation.
* A user needs front-channel or back-channel logout in production deployments using the default WildFly installation profile.

== Requirements

The existing OIDC logout functionality remains unchanged, but moves from PREVIEW to DEFAULT stability:

* RP-Initiated Logout remains supported.
* Front-Channel Logout remains supported.
* Back-Channel Logout remains supported.
* Existing configuration attributes remain available and unchanged in behavior:
** `logout-path`
** `post-logout-redirect-uri`
** `logout-callback-path`
** `logout-session-required`
** `back-channel-logout-session-invalidation-limit`
** `provider-jwt-claims-typ`

=== Changed Requirements

* The `elytron-oidc-client` logout capabilities should be available in DEFAULT stability and therefore in standard WildFly server/runtime usage.

=== Non-Requirements

* No new OIDC logout feature semantics are introduced in this promotion.
* No new configuration attributes are introduced.

=== Future Work

* Evaluate additional logout interoperability scenarios against providers beyond currently covered testing sets.

== Backwards Compatibility

No incompatibility is expected from this promotion.
The feature behavior and configuration model remain unchanged.

=== Default Configuration

No new default subsystem configuration values are required.
This is a stability promotion only.

=== Deployments

Existing deployments using OIDC logout options continue to work as-is.
Deployments no longer need PREVIEW stability solely to use OIDC logout support.

=== Interoperability

No new interoperability concerns are introduced by this stability promotion.

== Implementation Plan

* Promote OIDC logout support and related model/schema elements from PREVIEW to DEFAULT.
